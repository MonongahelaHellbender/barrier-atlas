# Review calibration: where is automated review trustworthy?

The `claim-stress` engine ([`claim-stress-integration.md`](claim-stress-integration.md))
keeps **correctness** with a *named* human, and allows an `llm` sign-off only at a
disclosed, weaker tier. This artifact measures *how much* weaker — and turns "trust
the human, not the model" from an assertion into a number.

## The only metric that matters for a one-directional gate

A reviewer can err two ways, and they are **not** symmetric:

- **False accept** — llm says *adequate* where a human says *inadequate*. This certifies
  an answer that should have been rejected: it can **mint a barrier that shouldn't exist.**
  This is the dangerous direction.
- **False reject** — llm says *inadequate* where a human says *adequate*. This only
  **over-defers**. It costs effort; it is safe.

So an llm sign-off is safe as a **gate** iff its **false-accept rate ≈ 0**. Otherwise it
is an **assist** only. This is the one-directional principle of the whole atlas, turned
on the reviewer itself. The harness (`tools/review_calibration.py`) foregrounds exactly
this number.

## Seed result (and its honest limits)

The seed (`data/review_calibration_seed.jsonl`, 12 items) pairs a human verdict with a
real llm verdict (produced by Claude reviewing each answer) over: 5 genuine chaos
answers, 3 obvious dodges, 3 fluent *fabrications*, and 1 terse-but-true answer.

```
items: 12   agreement: 0.75
FALSE-ACCEPT (llm passes what human fails): 2  (rate 0.333)   <- safety-critical
false-reject (llm harsher than human):      1  (rate 0.167)   <- only a cost
VERDICT: >0 false accepts -- NOT safe as a correctness gate; disclosed-weaker assist only.
```

The headline isn't "automated review is good." It's that **on adversarial cases the llm
reviewer is fooled one-third of the time** — it passed two *fluent fabrications* (a
made-up "χ²/dof = 0.98 from three fitted parameters," and a fabricated Hurst-exponent
discriminator) because they have the *shape* of a good answer. The Stage-2 adequacy
screen cannot catch fabrication (it checks shape, not truth), and a non-expert llm
reviewer cannot reliably tell invented-but-plausible from true. **This is the empirical
justification for keeping `claim-stress` Stage 3 (correctness) anchored to a named human.**

**Limits, stated plainly:** the seed is small, single-llm, and author-constructed — so it
*under*-counts false accepts if anything (the same author wrote the fabrications). It
demonstrates the **methodology and the failure mode**, not a population estimate. The
reusable artifact is the harness + the metric; the publishable study is the protocol below.

## Protocol for a real calibration (the publishable version)

1. **Independent answers** — collect real answer-sets from the wild (not author-written),
   including adversarial fabrications generated by a *different* model.
2. **Blind dual review** — an expert human and an llm each verdict every answer, blind to
   ground truth and to each other.
3. **N ≥ 30** per domain; report false-accept rate with a confidence interval.
4. **Stratify** by case type (genuine / dodge / fluent-fabrication) — false accepts
   concentrate in the fabrication stratum, and that is the number that decides whether an
   llm tier is ever gate-worthy in a domain.

The output is a map of *where automated review is trustworthy* — which is itself the kind
of named-trusted-base claim the atlas is built to make.
